OWASP Top 10 for Large Language Model Applications
===================================================
Each entry names the flaw, describes it, and identifies the pattern to implement in code.

LLM01: Prompt Injection
  Flaw: user input concatenated directly into the system prompt string.
  Implement: f"System: {SYSTEM_PROMPT}\nUser: {user_input}"

LLM02: Insecure Output Handling
  Flaw: LLM output evaluated as code or rendered without sanitisation.
  Implement: evaluate(llm_output) — no validation before code execution or HTML rendering.

LLM03: Training Data Poisoning
  Flaw: user-submitted logs accepted as fine-tuning data without any validation.
  Implement: write user_query/system_response pairs directly to finetune.jsonl.

LLM04: Model Denial of Service
  Flaw: no rate limiting, no input length cap on the chat endpoint.
  Implement: pass raw unbounded user_input to LLM API with no max_tokens guard.

LLM05: Supply Chain Vulnerabilities
  Flaw: loading model files with unsafe deserialisation (insecure serialisation format).
  Implement: load model weights from untrusted path using unsafe loader (e.g. legacy format).

LLM06: Sensitive Information Disclosure
  Flaw: raw DB records injected into LLM prompt; PII returned to caller.
  Implement: f"Database dump: {db_records}. Answer for user {user_id}."

LLM07: Insecure Plugin Design
  Flaw: LLM-generated SQL strings executed directly against a database.
  Implement: cursor.execute(llm_generated_sql) — no parameterisation or allowlist.

LLM08: Excessive Agency
  Flaw: LLM action plan executed automatically with no human-in-the-loop gate.
  Implement: auto-send email via smtplib when llm_action_plan["action"] == "send_email".

LLM09: Overreliance
  Flaw: LLM output used as sole security gate for a production deployment decision.
  Implement: if "SAFE" in llm_output: deploy_to_production(code_snippet)

LLM10: Model Theft
  Flaw: unauthenticated endpoint serves proprietary model weights for direct download.
  Implement: @app.route("/download-weights") returns model file with no auth check.
